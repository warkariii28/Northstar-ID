Why it feels different

More than a plain tutorial clone

The app now has a stronger brand direction, a refined UI system, and a cleaner onboarding flow that makes it easier to present as your own project.

Register Users create accounts with validated username, email, and password fields.
Authenticate Passwords are hashed and verified through Bcrypt during login.
Persist Flask-Login keeps the session active and user state available across requests.

Secure foundation

Passwords are never stored in plain text

`Flask-Bcrypt` generates the password hash on registration and checks it on login.

Clean validation

The form layer blocks common bad input

Email format, password length, confirmation matching, and duplicate users are handled before bad data lands in the database.
